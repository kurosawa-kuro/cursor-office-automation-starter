# VOICEPEAK 連携 成功ノウハウ（ナレーション動画の作り方）

スライド（[`90_動画スライド.md`](./90_動画スライド.md)）＋ ナレーション音声（VOICEPEAK）から動画を作るための、**実際に通った手順**の記録。

---

## 0. ファイルの役割

| ファイル | 役割 |
|---|---|
| `90_動画スライド.md` | Marp スライド本体（概念＋スクショ50枚＋presenter notes） |
| `90_動画スライド.pptx` | 上記から書き出した PowerPoint（50枚・各スライドは画像埋め込み） |
| `90_動画スライド-ナレーション.txt` | **VOICEPEAK 読み込み用**（1スライド=1ブロック・`===` 区切り・カタカナ読み） |
| `90_動画スライド-ナレーション.md` | 人間用の読み台本（見出しあり・記号あり） |
| `91_動画化メモ.md` | 公開前チェック（個人情報マスク等） |

---

## 1. ★最重要：VOICEPEAK のテキスト読み込み設定

`90_動画スライド-ナレーション.txt` を読み込むときの設定（マニュアル 3-8）:

| 項目 | 設定 |
|---|---|
| 改行で分割 | **OFF** |
| カスタム分割文字 | **`===` を指定して ON** |
| 空行を除外 | ON（任意） |

→ これで **1ブロック = 1スライド**（全50ブロック）になり、スライドと完全同期する。

**失敗例（やりがちなNG）**: 既定の「改行で分割 ON」のまま読み込むと、**1文ごとに細切れのブロック**になり破綻する。必ず改行分割を OFF にし、`===` をカスタム分割文字に指定する。

---

## 2. ナレーション .txt の作り方ルール（誤読・崩れ防止）

VOICEPEAK は読み込んだテキストを**そのまま読み上げる**ため、以下を守る:

1. **Markdown 記号を入れない** — `#` `**` `>` `---` 表 `` ` `` は読み上げ・ブロック崩れの原因。純粋なテキストにする。
2. **1スライド = 1ブロック**。ブロック構造は「見出し行 ＋ 本文（複数文を1行に連結）」、ブロック間は **`===`** で区切る。
3. **英語・記号はカタカナ読みに置換**（VOICEPEAK の誤読・英語読みを防ぐ）:
   - Cursor→カーソル / Windows→ウィンドウズ / Excel→エクセル / Python→パイソン / AI→エーアイ
   - ZIP→ジップ / Chat→チャット / README→リードミー / Open Preview→オープン プレビュー / Code→コード
   - GitHub→ギットハブ / Google→グーグル / Apple→アップル / Free Plan→フリープラン / -master→マスター
   - cursor.com→カーソル ドット コム / 数字は読みを固定（例: 15分→十五分）
   - 全角ダブルクォート `“ ”` → 「 」
4. **見出し行もブロック内＝読み上げ対象**。動画で読ませたくない見出しは、VOICEPEAK 読み込み後に各ブロックの**先頭行を削除**する（または見出しを `◆` 等の短い記号にする運用）。
5. **スライドとブロックは順序を1対1に保つ**（差し込み・削除したら両方を直す）。

---

## 3. スライド ⇔ ナレーションのズレ確認（コマンド）

`90_動画スライド.md` のスライド見出しと `.txt` のブロック見出しを並べて照合する:

```bash
cd docs/教育資料
awk '/^---$/{f=1;next} f&&/^#+ /{sub(/^#+ /,"");print;f=0}' 90_動画スライド.md > /tmp/s.txt
awk 'BEGIN{p=1} /^===$/{p=1;next} p{print;p=0}' 90_動画スライド-ナレーション.txt > /tmp/n.txt
echo "slides=$(wc -l < /tmp/s.txt) narration=$(wc -l < /tmp/n.txt)"
paste -d'|' /tmp/s.txt /tmp/n.txt | cat -n
```

→ 件数が一致し、各行が同じスライドを指していれば OK（2026-05 時点で 50＝50 で一致確認済み）。

## 4. スライド → PowerPoint（PPTX）書き出し

Marp CLI で `90_動画スライド.md` から PPTX を生成:

```bash
cd docs/教育資料
CHROME_PATH=/usr/bin/chromium-browser \
  npx -y @marp-team/marp-cli@latest 90_動画スライド.md --pptx --allow-local-files -o 90_動画スライド.pptx
```

- `--allow-local-files`: `images/` のスクショを埋め込むのに必須
- 生成 PPTX は **各スライドが画像**（文字編集不可）。スライドショー・動画化はこれで十分
- **文字編集したい場合**は `--pptx-editable`（要 LibreOffice / `soffice`）
- **日本語フォント**: 書き出し環境に日本語フォントが必要。`90_動画スライド.md` の style に `IPAGothic` を含めてある（Linux）。豆腐（□）になる場合は `Noto Sans JP` / `Yu Gothic` を入れて再書き出し

---

## 5. 動画化の流れ

1. VOICEPEAK で `.txt` を上記設定（§1）で読み込み → 各ブロック（=スライド）の音声を生成・WAV 書き出し
2. PowerPoint で `90_動画スライド.pptx` を開く → 「スライドショーの記録」で各スライドにナレーション音声を乗せる → 動画（mp4）にエクスポート
   - もしくは スライド画像 ＋ WAV を動画編集ソフトに並べて合成
3. **想定尺**: 全体で約 7〜8 分

## 7. ハマりどころ早見

| 症状 | 原因 / 対処 |
|---|---|
| ブロックが1文ごとに細切れ | 改行分割が ON のまま。**OFF にして `===` をカスタム分割文字に指定** |
| 英語が英語読み・変な読み | テキストがカタカナ化されていない。§2-3 に従い置換 |
| 見出しまで読み上げられる | 見出しはブロック内＝読まれる。不要なら読み込み後に先頭行を削除 |
| スライドと音声がズレる | スライドとブロックの数・順序不一致。§3 のコマンドで照合 |
| PPTX の日本語が □（豆腐） | 書き出し環境に日本語フォント無し。IPAGothic / Noto Sans JP を入れて再書き出し |
