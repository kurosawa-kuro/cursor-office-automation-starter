# 91. デモ収録シナリオ（逐語ショット台本）

[`90_動画台本.md`](./90_動画台本.md) の `🎥 画面録画` プレースホルダ（デモ①〜⑤）を、**そのまま撮れるレベル**まで具体化した収録台本です。
ナレーション（話す内容）は 90、本ファイルは **映す内容（操作）** を担当します。これは収録用の制作ドキュメントで、スライドではありません（00〜04 の学習パスとも別レイヤー）。

> 原則: クリック1つ・入力文字列・期待画面をショット単位で逐語化。プロンプト本文は [`../../src/office-task/README.md`](../../src/office-task/README.md) が正本。

---

## 0. 収録の前提（最初に1回だけ整える）

| 項目 | 設定 | 理由 |
|---|---|---|
| 画面解像度 | 1920×1080（16:9） | 90 のスライドと同じ比率 |
| Cursor ズーム | `Ctrl` + `=` を 2〜3 回（UI を拡大） | 動画で文字が読める |
| テーマ | Light（明るいテーマ） | 録画・圧縮で潰れにくい |
| 表示言語 | 日本語（`Configure Display Language`） | 受講者の画面と一致 |
| ウィンドウ | 最大化、通知 OFF、不要タブを閉じる | 余計な物を映さない |
| アカウント | 収録用の使い捨て Hobby アカウント | 個人情報を映さない |

> 🔒 **共通NG（全デモ）**: 実在のメールアドレス・電話番号・SMS コード・社内データ・トークンを画面に映さない。映る場合はモザイク or 再現用ダミーに差し替え。

## §1. デモ①：Cursor をインストール

- **目的**: 公式サイトから入手 → インストール完了 → 起動、までを見せる
- **事前状態**: Cursor 未インストール。ブラウザ（Edge）を開いた状態

| # | 操作 | 入力 / クリック | 期待画面 |
|---|---|---|---|
| 1 | アドレスバーに入力 | `https://cursor.com/` | 公式トップ |
| 2 | クリック | 「Download for Windows」ボタン | `.exe` のダウンロード開始 |
| 3 | ダウンロードした exe を実行 | `Cursor Setup-x.x.x.exe` をダブルクリック | インストーラ起動（UAC は「はい」） |
| 4 | 進める | 同意 → 既定のまま `Install` → `Finish` | Cursor が自動起動 |

- **見せる要点**: 「Download for Windows」ボタンにマウスを置いて一拍。インストールは早送り可
- **カット点**: ダウンロード待ち・インストール進捗バーは**早送り or カット**
- **撮り直し保険**: exe 実行時の「Windows によって PC が保護されました」が出たら、別テイクで「詳細情報→実行」も撮っておく（トラブル説明用の差し込み素材）
- 詳細手順の正本: [`02_環境構築ガイド.md`](./02_環境構築ガイド.md) §①②

## §2. デモ②：サインイン（安心ポイントを見せる）

- **目的**: 無料・カード不要で始まることを画面で示す
- **事前状態**: Cursor 初回起動の Welcome 画面
- **⚠️ 個人情報注意**: メール入力欄・SMS コードは**ダミー表示 or モザイク**。実アカウントの作成画面を撮るなら、後でぼかす前提で。

| # | 操作 | 入力 / クリック | 期待画面 |
|---|---|---|---|
| 1 | クリック | `Sign up`（または `Sign in`） | ブラウザにサインインページ |
| 2 | 選択 | 「メールで続行」（GitHub は使わない＝任意であることを口頭で） | メール入力欄 |
| 3 | 入力（ダミー） | `demo@example.com`（実物は映さない） | 認証コード送信 |
| 4 | SMS 認証 | 届いたコードを入力（**コードは映さない**） | 認証完了 |
| 5 | 戻る | ブラウザ「Cursor を開きますか？」→ 許可 | Cursor 本体に戻る |
| 6 | 確認 | 右下のアカウント名を指す | サインイン済み表示 |

- **見せる要点**: テロップで `クレカ不要 / SMSのみ / GitHub不要 / Privacy Modeで学習オフ`（90 のスライド②と一致）
- **カット点**: メール送受信の待ち時間はカット
- **任意ショット**: `Cursor Settings → Privacy` を開いて `Privacy Mode` を ON にする様子（数秒）。実業務データを扱う層への安心材料
- 詳細手順の正本: [`02_環境構築ガイド.md`](./02_環境構築ガイド.md) §③④

## §3. デモ③：教材フォルダを開く

- **目的**: 教材リポジトリを取得して Cursor で開くまで
- **事前状態**: サインイン済み・フォルダ未取得

| # | 操作 | 入力 / クリック | 期待画面 |
|---|---|---|---|
| 1 | クリック | Welcome 画面の `Clone repo` | URL 入力欄 |
| 2 | 貼り付け | `https://github.com/kurosawa-kuro/cursor-office-automation-starter.git` | URL が入る |
| 3 | 保存先選択 | 例: `C:\dev\` | クローン進行 |
| 4 | 信頼ダイアログ | `Yes, I trust the authors` | フォルダが開く |
| 5 | 確認 | 左サイドバーの `src/office-task/` を開いて指す | `README.md` と `data/` が見える |

- **見せる要点**: 左ツリーに `src/office-task/README.md`（プロンプト集）と `data/`（サンプル）が入っていること
- **カット点**: クローンの進捗はカット
- **撮り直し保険**: `Authentication failed` が出たら URL を再確認（パブリックなので認証不要）。`git: command not found` は Clone repo を使えば回避
- 詳細手順の正本: [`03_ハンズオン.md`](./03_ハンズオン.md) §①〜③

## §4. デモ④：Chat に最初のお願い（お礼メール）

- **目的**: 「日本語で頼む → 数秒で下書き」の成功体験を見せる
- **事前状態**: 教材フォルダを開いた状態
- **使うプロンプト**: プロンプト1（商談後のお礼・提案メール／入力データ不要）

| # | 操作 | 入力 / クリック | 期待画面 |
|---|---|---|---|
| 1 | Chat を開く | `Ctrl + L` | 右ペインに Chat 欄 |
| 2 | プロンプトを貼って送信 | 下の囲みを貼り付け → Enter | AI が生成開始 |
| 3 | 結果を見せる | スクロールして全体を映す | 件名〜署名のメール下書き |

貼り付けるプロンプト（逐語）:

```
IT エンジニア派遣の営業として、初回商談のお礼メール（件名・本文・署名欄）を日本語で作ってください。
- 宛先: クライアント A 社 採用担当 田中様
- 内容: 本日の Java バックエンドエンジニアの募集ヒアリングのお礼 + 候補エンジニア 1 名を後日提案する旨
```

- **期待アウトプット**: 件名 / 宛名 / 時候の挨拶 / 本文 / 結び / 署名欄 を含む自然な日本語ビジネスメール
- **見せる要点**: 入力は日本語の箇条書きだけ → 整ったメールが返る対比。生成が走る数秒はそのまま見せて「待つだけ」感を出す
- **カット点**: 生成が長引いたら中盤を軽く早送り
- **撮り直し保険**: 応答に `pip install` 等が混ざったら別テイク（このキットのルールでは出ない想定。出たら 04 §8.D の追記文で再依頼する様子も差し込み素材に）
- プロンプト正本: [`../../src/office-task/README.md`](../../src/office-task/README.md) 「プロンプト1」

## §5. デモ⑤：データを読ませて集計

- **目的**: `@file` でファイルを読ませ、表で返るのを見せる（Excel 関数なしで集計）
- **事前状態**: 教材フォルダを開いた状態。Chat 欄が開いている
- **使うデータ**: `src/office-task/data/新規 データ1.csv`（エンジニア一覧）

| # | 操作 | 入力 / クリック | 期待画面 |
|---|---|---|---|
| 1 | `@` を打つ | Chat 欄で `@` 入力 | ファイル候補が出る |
| 2 | ファイル選択 | 候補から `新規 データ1.csv` を選ぶ | `@file ...新規 データ1.csv` が入る |
| 3 | 続けて指示を入力 → 送信 | 下の囲みの後半を入力 → Enter | AI が CSV を読んで集計 |
| 4 | 結果を見せる | 返ってきた表を映す | Markdown 表（スキル別人数・平均単価） |

貼り付けるプロンプト（逐語・`@file` 込み）:

```
@file src/office-task/data/新規 データ1.csv の内容を読み、まずこのファイルが何のデータかを1行で要約し、続いて主力スキル別の人数と月単価(monthly_rate)の平均を Markdown 表で出してください。
```

- **期待アウトプット**: 「これはエンジニア一覧です」の要約 + スキル別人数 + monthly_rate 平均の表
- **見せる要点**: `@` を打つと候補が出てクリックで選べること（ファイル名を覚えなくてよい）
- **発展ショット（任意）**: 2 ファイル版の集計（プロンプト4：`新規 データ1.csv` + `新規 データ2.csv`）を「もっと本格的にもできる」として一瞬だけ見せる
- **カット点**: 生成中盤は早送り可
- プロンプト正本: [`../../src/office-task/README.md`](../../src/office-task/README.md) 「プロンプト4」

## 通し収録順とつなぎ

```
  demo01_install → demo02_signin → demo03_open → demo04_chat_mail → demo05_csv
       │（Phase 1：一度だけ）        │（Phase 2：毎回・ここがメイン）
       └────────────────────────────┘
```

- **推奨**: ①〜③ は一気に通し収録（環境がリセットされるため）。④⑤ はフォルダを開いた同一セッションで連続収録
- 各デモの頭で 1 秒静止（編集でスライド→画面の切り替えがしやすい）
- 末尾は次のデモに自然につながるよう、画面を動かさず止める
