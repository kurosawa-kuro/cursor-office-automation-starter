# 91. 動画化メモ（書き出し・公開前チェック）

動画は **ライブ画面録画をせず**、[`90_動画スライド.md`](./90_動画スライド.md) のスライド（02/03 で撮影済みのスクリーンショット ＋ ナレーション）を**流し込んで**作ります。
本ファイルは、その **書き出し手順と公開前チェック**をまとめた制作メモです（旧・逐語ショット台本は 90 のスライドに統合したため廃止）。

> 操作の正本は 02/03（画像つき手順）、台本の正本は 90（スライド＋presenter notes のナレーション）。

---

## 1. 動画の作り方（スライドショー型）

```
  90_動画スライド.md（Marp）
        │  Marp で書き出し
        ▼
  PDF / PPTX / 画像（1スライド=1枚）
        │  ナレーションを各スライドに重ねる
        ▼
  動画（mp4）
```

- **書き出し**: VS Code / Cursor の Marp 拡張、または `npx @marp-team/marp-cli 90_動画スライド.md --pptx`（PDF / 画像も可）
- **ナレーション**: 各スライドの presenter notes（`<!-- -->`）の文をそのまま読む。`所要：N秒` が目安
- **作り方の例**:
  - PPTX に書き出し → PowerPoint の「スライドショーの記録」で音声を吹き込み → 動画にエクスポート
  - または 画像（PNG）に書き出し → 動画編集ソフトに並べ、ナレーション音声を重ねる
- **想定尺**: 全体で **約 7〜8 分**（presenter notes の所要秒の合計が目安）

## 2. 公開前チェック（最重要：個人情報のマスク）

撮影済みスクショには **実 PC の情報が映り込んでいます**。公開前に必ずマスク（ぼかし / トリミング / 差し替え）してください。

| 映り込み | 該当スクショ（例） | 対処 |
|---|---|---|
| Windows ユーザー名（`owner`）・ローカルパス | `open-04-extract-dialog` / `prompt-12-write-file-request`（`C:\Users\owner\...`） | ユーザー名部分をぼかす or 一般化 |
| 無関係な個人フォルダ名 | `open-02-downloaded-zip` / `open-05-zip-and-folder` / `open-06-delete-zip`（`環境認識` `ML-basic` `Video_change` 等） | 余白をトリミング、または一覧部分をぼかす |
| 最近開いたプロジェクト一覧・接続先 | `open-08-opened` / `install-08-launched` / **`signin-01-settings-gear`**（`永続保存` `repos [WSL]` `.ssh` 等） | Welcome 画面の「Recent / 接続先」一覧をぼかす |
| `.ssh` 等の機微フォルダ | `open-02-downloaded-zip` 左ペイン | トリミング推奨 |
| **実在のメールアドレス**（アカウント名） | **`signin-02-account-open`**（`kurosawa…@gmail` が表示） | メールアドレス部分をぼかす。**02_環境構築ガイド §③ にも掲載**しているので、公開教材として配る前に要マスク |

> ✅ サンプルデータ（`山田太郎` のスキルシート等）は**架空**なので、そのまま表示して OK。
> 🔒 サインイン方法の選択画面（`signin-03-method`：Google / GitHub / Apple / メール）は個人情報を含まないのでそのまま可。**メール入力・SMS コードの画面は映さない**こと。

## 3. 仕上げの方針（任意）

- **テロップ**: 安心ポイント（クレカ不要 / SMS のみ / GitHub 不要 / Privacy Mode）はスライド2と一致させる
- **ズーム**: 重要な小さい UI（ボタン・右クリックメニュー）は、編集で軽く拡大すると非エンジニアに親切
- **間（ま）**: スライド切替の頭で 0.5〜1 秒の静止を入れると聞き取りやすい
- **BGM**: 入れる場合は小音量。ナレーションを邪魔しない
